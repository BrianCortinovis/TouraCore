Security — OWASP & secrets

🔴 Critical

CRITC1 — Bundle checkout accetta prezzi client-controllati
apps/web/src/app/api/v1/bundles/route.ts:23-31, 106, 187
Endpoint POST /api/v1/bundles totalmente unauthenticated, accetta unitPriceCents + vatRate dal body e li passa verbatim a Stripe Checkout. PoC: curl -d '{"unitPriceCents":1,...}' → camera €500 pagata €0.01.
Fix: ricalcolare server-side da entityId+config via pricing engine. Mai trustare amount client.
CRITC2 — Tourist tax PaymentIntent: amount client-controlled
apps/web/src/app/checkin/[token]/actions.ts:61-94
createTouristTaxPaymentIntentAction legge amountCents dal client senza ricalcolo da tourist_tax_rates.
Fix: ricalcolare da tourist_tax_rates × adults+children × nights per reservation_id.
CRITC3 — Bundle checkout bypassa Stripe Connect (soldi a TouraCore)
apps/web/src/app/api/v1/bundles/route.ts:177-204
Crea Checkout senza transfer_data/application_fee_amount. Tutti i fondi finiscono nell'account platform → viola regola "TouraCore mai banca".
Fix: usare buildConnectChargeParamsSafe come fa /api/public/booking/checkout/route.ts.
CRITC4 — API key generate con Math.random() (predicibile)
apps/web/src/app/api/public/booking/_shared.ts:111
generateApiKey usa Math.random() per prefix + secret 32-char. PRNG V8 non crypto-safe → reconstruct state.
Fix: crypto.randomBytes(16).toString('hex').
CRITC5 — Smart-lock PIN porte fisiche con Math.random()
apps/web/src/app/api/cron/lock-pins/route.ts:14 + apps/web/src/lib/lock-providers.ts:27
PIN 6-cifre per smart lock fisici generati con Math.floor(Math.random()*10). Spazio 10⁶ + biased.
Fix: crypto.randomInt(100000, 1000000).
