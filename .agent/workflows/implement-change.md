# Implement Change

Usa questo workflow per implementare una modifica dopo che il piano e chiaro.

Segui questi passi:

1. applica il minimo cambiamento necessario
2. mantieni il codice coerente con pattern esistenti
3. evita refactor larghi se il task e locale
4. aggiorna i test o aggiungili se il comportamento cambia
5. aggiorna la documentazione se il cambiamento e stabile e non solo locale
6. prepara un closeout finale con test e rischi

Nel closeout includi sempre:

- file toccati
- comandi eseguiti
- esito dei check
- follow-up consigliati
