# Close Task

Usa questo workflow prima di considerare finito un task.

Checklist:

1. conferma che il codice e coerente con i pattern del repo
2. esegui i check minimi rilevanti o spiega gli skip
3. verifica se la documentazione va aggiornata
4. riassumi in modo breve ma completo

Formato del closeout:

- scope del task
- file toccati
- comandi eseguiti
- test passati, falliti o saltati
- rischi residui
- eventuali prossimi passi
