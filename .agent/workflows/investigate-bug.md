# Investigate Bug

Usa questo workflow per debug o regressioni.

Segui questi passi:

1. descrivi il bug in modo verificabile
2. individua area, package o route piu probabili
3. formula 2-3 ipotesi concrete
4. verifica le ipotesi con codice, test, log o browser
5. proponi il fix piu piccolo possibile
6. indica come prevenire regressioni con test o verifiche

Output atteso:

- sintomo
- ipotesi
- causa probabile
- fix proposto o implementato
- check di regressione
