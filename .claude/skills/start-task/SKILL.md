# Start Task

Segui questa procedura:

1. leggi `CLAUDE.md`
2. apri solo i documenti pertinenti in `docs/`
3. individua package, route, verticale o migrazione coinvolta
4. cerca implementazioni simili nella repo prima di proporre nuove astrazioni
5. scrivi un piano breve e locale al task
6. definisci i check minimi rilevanti prima di modificare codice

Nel piano finale includi sempre:

- area impattata
- rischio principale
- file o cartelle probabili
- comandi di verifica previsti
